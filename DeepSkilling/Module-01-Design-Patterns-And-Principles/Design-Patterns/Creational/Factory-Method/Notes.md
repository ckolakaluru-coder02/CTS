\# Factory Method Pattern



The Factory Method Pattern is a creational design pattern used to create objects without exposing the object creation logic to the client.



The client uses a factory to get the required object.



\## Real-time Example



A document factory can create different types of documents like Word, PDF, and Excel.



\## Advantages



\- Hides object creation logic

\- Reduces tight coupling

\- Makes code flexible

\- Easy to add new object types
