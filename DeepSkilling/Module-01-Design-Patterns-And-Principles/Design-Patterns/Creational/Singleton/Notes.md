\# Singleton Pattern



The Singleton Pattern is a creational design pattern that ensures only one object of a class is created.



It provides a global access point to that object.



\## Real-time Example



A logger class is commonly implemented using Singleton because one common logger instance can be used throughout the application.



\## Advantages



\- Only one instance is created

\- Saves memory

\- Provides global access

\- Useful for logging, configuration, and database connection management
