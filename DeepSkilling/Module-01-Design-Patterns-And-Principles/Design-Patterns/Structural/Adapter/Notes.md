\# Adapter Pattern



The Adapter Pattern is a structural design pattern that allows incompatible interfaces to work together.



It acts as a bridge between two different interfaces.



\## Real-time Example



A mobile charger adapter allows a charger with one type of plug to connect to a different socket.



\## Advantages



\- Helps incompatible classes work together

\- Improves code reusability

\- Avoids changing existing code

\- Makes integration easier
