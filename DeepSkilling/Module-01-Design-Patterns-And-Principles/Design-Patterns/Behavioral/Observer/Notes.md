\# Observer Pattern



The Observer Pattern is a behavioral design pattern where one object notifies other dependent objects when its state changes.



The object that sends updates is called the Subject.  

The objects that receive updates are called Observers.



\## Real-time Example



YouTube channel subscription is an example of Observer Pattern.  

When a channel uploads a new video, all subscribers get notified.



\## Advantages



\- Supports loose coupling

\- Useful for event handling

\- One object can notify many objects

\- Improves communication between objects
