\# Command Pattern



The Command Pattern is a behavioral design pattern that turns a request into an object.



It separates the object that sends the request from the object that performs the action.



\## Real-time Example



A remote control is an example of Command Pattern.  

Pressing a button sends a command to turn a device on or off.



\## Advantages



\- Separates sender and receiver

\- Makes code flexible

\- Supports undo/redo operations

\- Useful for menu actions and remote controls
