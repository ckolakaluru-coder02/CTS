\# Single Responsibility Principle



The Single Responsibility Principle says that a class should have only one reason to change.



It means one class should perform only one responsibility.



\## Example



If a class is used to generate reports, it should only generate reports.  

Saving the report to a file or database should be handled by another class.



\## Advantages



\- Code becomes easy to understand

\- Code becomes easy to test

\- Changes in one responsibility do not affect other responsibilities

\- Improves maintainability
