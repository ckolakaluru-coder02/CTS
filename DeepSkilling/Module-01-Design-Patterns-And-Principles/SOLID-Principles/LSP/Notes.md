\# Liskov Substitution Principle



The Liskov Substitution Principle says that objects of a parent class should be replaceable with objects of child classes without affecting the program.



\## Example



If a method expects a Shape object, then Circle and Rectangle objects should work correctly in that method.



\## Advantages



\- Improves inheritance design

\- Avoids unexpected behavior

\- Makes code more reliable

\- Supports polymorphism correctly
