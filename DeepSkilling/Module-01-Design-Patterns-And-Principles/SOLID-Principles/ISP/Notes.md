\# Interface Segregation Principle



The Interface Segregation Principle says that a class should not be forced to implement methods it does not use.



Instead of one large interface, we should create smaller and specific interfaces.



\## Example



A simple printer should only implement printing.  

A scanner should only implement scanning.



\## Advantages



\- Avoids unnecessary methods

\- Reduces code complexity

\- Makes interfaces clean

\- Improves maintainability
