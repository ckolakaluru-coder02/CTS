\# Open/Closed Principle



The Open/Closed Principle says that software entities should be open for extension but closed for modification.



It means we should add new functionality without changing existing tested code.



\## Example



If we want to add new payment methods, we should create new classes instead of modifying the existing payment code.



\## Advantages



\- Reduces risk of breaking existing code

\- Makes code flexible

\- Supports future extensions

\- Improves reusability
